## Why

Homespun's prompt catalogue (DB-backed `AgentPrompt` records + `{{placeholder}}` template engine) duplicates what Claude Code already provides via filesystem skills. OpenSpec installs 8 skills under `.claude/skills/openspec-*/SKILL.md` — each identified by `metadata.author: openspec` in frontmatter. These skills are schema-agnostic: they call `openspec status` and `openspec instructions` at runtime to discover artifacts from whatever schema is active. Custom schemas do not generate new skills — the set is fixed at 8.

Replacing DB-backed prompts with skill-based dispatch eliminates ~800 LOC of server code plus ~4,000 LOC of web code, achieves CLI/app parity, and puts prompt definitions under version control. Since commands are being deprecated from Claude Code in favour of skills, skills are the sole dispatch mechanism.

## What Changes

- **Add** skill discovery service that walks the active clone's `.claude/skills/**/SKILL.md`, reads frontmatter, and returns two collections:
  - **OpenSpec skills**: hard-coded list of 8 skill names. Not dynamically discovered. Surfaced in the OpenSpec agent-runner tab.
  - **Homespun prompt skills**: identified by `homespun: true` in SKILL.md frontmatter. These are user-authored custom prompt replacements surfaced in the Task Agent tab.
- **Add** Homespun skill frontmatter convention: custom prompt skills declare `homespun: true`, `homespun-mode: plan|build`, and `homespun-args` with `kind: issue|change|phase-list|free-text` in their SKILL.md frontmatter so the UI can build the right input per skill. No sidecar needed — Homespun skills are user-authored and not regenerated by OpenSpec.
- **Modify** agent dispatch: app reads the skill's `SKILL.md` body, appends composed args, and sends the combined text as the session's `initialMessage`. No templating engine on the server side. The skill body IS the prompt; the app just composes the args portion.
- **Remove** `Features/ClaudeCode/Services/AgentPromptService.cs`, `AgentPromptsController.cs`, `DefaultPromptDefinition.cs`, `DefaultPromptsInitializationService.cs`, `default-prompts.json` and the `AgentPrompt` / `PromptContext` / `Mode` / `Category` / `SessionType` types from `Homespun.Shared`.
- **Remove** the `features/prompts/` React slice (7 components, 13 hooks, ~4,000 LOC).
- **Keep** `MiniPromptService` — still used for generating branch names.
- **Propagation**: skills in the server's "main clone" are treated the same as `.fleece/`: copied into new branch clones at creation; merged back when branches merge. In-flight branches use their own committed copy; edits on main do not auto-propagate.

## Capabilities

### New Capabilities
- `skills-catalogue`: Skill-based dispatch with sidecar metadata for custom prompts, OpenSpec skill discovery via frontmatter author field, UI surfacing of all available skills and per-skill argument kinds.

### Removed Capabilities
- `prompt-catalogue`: entirely superseded.

## Impact

- **Backend**: delete `Features/ClaudeCode/Services/AgentPrompt*`, `DefaultPrompt*` (~4 files, ~800 LOC). Add `SkillDiscoveryService` and an `ISkillDescriptor` contract under `Features/ClaudeCode/` (~250 LOC new). Update `IssuesAgentController` to read skill body + compose args as initial message. Drop `AgentPrompt`/`PromptContext`/Mode/Category/SessionType from `Homespun.Shared`.
- **Frontend**: delete `features/prompts/` slice (~4,000 LOC). Add skill-picker UI in the Task Agent tab (~600 LOC). Surface all skills in session chat window (~300 LOC).
- **Tests**: remove existing web tests for prompts slice (14 files). Add skill-discovery unit tests + integration test for session dispatch with skill body composition.
- **Migration**: none — `AgentPrompt` table can be dropped without user action because behaviour is replaced by filesystem skills.
- **Risk**: `openspec update --force` overwrites OpenSpec-generated `SKILL.md` files. This is irrelevant because OpenSpec skills are hard-coded in Homespun — their metadata is not derived from frontmatter. Homespun-authored skills are not regenerated by OpenSpec, so their frontmatter is stable. Custom schemas do not generate new skills — set fixed at 8.
